merger module¶

class merger.BaseMerger[source]¶

Bases: object

All mergers have to extend this class.

Must declare:

get_supported_software()

merge()

Can declare:
get_priority()

See mergers.patternmerger.Merger for an example implementation.

apply(software, theme)[source]¶
get_priority()[source]¶

Priority of the merger. If multiple mergers claim support for a software, the one with the highest priority is used.

Default:0
get_supported_software()[source]¶

Tells us which software is supported by this merger. Must be overridden by inheriting classes. Example: mergers.patternmerger.Merger.get_supported_software()

Returns:dict - { “software”: {“theme-filename” : [“real”, “filenames”]} }
merge(software, name, oldconfig, newconfig)[source]¶

When a theme is applied, this is called for each file we declared to support in self.files and that is included in the theme

Parameters:
  • name – the name of the file to merge in the theme
  • oldconfig – the path to the original config
  • newconfig – the path to the config to merge
Returns:

a list containing the patched config line by line
