merger module¶

class merger.BaseMerger[source]¶

Bases: object

All mergers have to extend this class.

Things to declare:
self.files = { name : [path1, path2, ...], ... }

path1, path2, ...: possible places where the supported software may store the specific config file. expanduser is called on them, so you can use paths like ~/.bla/blubb

name: the name of the file in a theme.

Example: 
self.files = {"config": [ "~/.i3/config"),
                path.join(xdg_config_home, "i3/config") ] }
merge(self, name, oldconfig, newconfig)¶

When a theme is applied, this is called for each file we declared to support in self.files and that is included in the theme

Parameters:
  • name – the name of the file to merge in the theme
  • oldconfig – the path to the original config
  • newconfig – the path to the config to merge
Returns:

a list containing the patched config line by line

See mergers.i3.Merger for an example implementation.
